Featured Projects

A4R Multi-Agent Architecture

Role: Core Architect & Developer (Holos Platform Challenge)

Background: Holos platform native Planning+Routing mechanism stateless, cannot mid-process ask, difficulty handling uncertain intent demands (IIUN).

Architecture Design (A4R): Proposed "intelligent preprocessing" scheme, constructed Guide, Validate, Discern three governance intelligent agents. GuideAgent as flow entry interacts with user multi-round, successfully converts ambiguous IIUN into platform executable deterministic task. Strictly adheres to A2A protocol and intelligent agent SEO norms, via Agent Card configuration ensures correct flow routing, and embeds ValidateAgent in execution flow for QA validation, solving platform inability for multi-round interaction.

Engineering Implementation: Independent scheduling: self-built PlanRoutingAgent takes over routing logic, achieves platform-independent end-to-end "planning-routing" workflow. Intelligent deduplication: designed "task fingerprint" algorithm extracts task core description, effectively solves platform splitting one task causing LLM repeated call problem. Robustness and debugging: designed "LLM + rule engine" dual-track degradation strategy, ensures system availability when API fails; established unified logging norm (contains contextId and taskId), realizes full-link tracking and fast debugging in 8-agent concurrent scenario.
